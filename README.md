# CRM Заявки

Минималистичная CRM для малой компании: заявки клиентов на сайте (GitHub Pages), хранение в Google Sheets, уведомления в Telegram.

## Стек

- **Фронтенд:** HTML, CSS, JavaScript → GitHub Pages
- **Бэкенд:** Google Apps Script + Google Sheets
- **Уведомления:** Telegram Bot API

## Структура

| Файл | Назначение |
|------|------------|
| `index.html` | Главная страница CRM |
| `style.css` | Стили интерфейса |
| `script.js` | Логика формы, таблицы, поиска, API |
| `config.js` | URL веб-приложения Apps Script |
| `Code.gs` | Серверный код (вставить в проект Apps Script) |
| `README.md` | Документация |

## Быстрый старт

### 1. Google Таблица

1. Создайте [новую Google Таблицу](https://sheets.google.com).
2. **Расширения → Apps Script**.
3. Удалите содержимое `Code.gs` и вставьте код из файла `Code.gs` этого репозитория.
4. Сохраните проект (Ctrl/Cmd + S).
5. В редакторе выберите функцию `setupSheet` → **Выполнить** → разрешите доступ.
   - Создастся лист **Заявки** с колонками: `id`, `name`, `phone`, `comment`, `createdAt`.

### 2. Telegram (опционально, но рекомендуется)

1. Создайте бота через [@BotFather](https://t.me/BotFather), получите **токен**.
2. Узнайте **Chat ID** (личный чат или группа):
   - Напишите боту сообщение.
   - Откройте `https://api.telegram.org/bot<TOKEN>/getUpdates` и найдите `"chat":{"id":...}`.
3. В Apps Script: **Проект → Свойства скрипта** → добавьте:

   | Свойство | Значение |
   |----------|----------|
   | `TELEGRAM_BOT_TOKEN` | токен бота |
   | `TELEGRAM_CHAT_ID` | ID чата |

При новой заявке бот отправит:

```
🆕 Новая заявка

Имя: ...
Телефон: ...
Комментарий: ...
```

### 3. Публикация веб-приложения Apps Script

1. **Развернуть → Новое развертывание**.
2. Тип: **Веб-приложение**.
3. Параметры:
   - Выполнять как: **Я**
   - У кого есть доступ: **Все**
4. Скопируйте URL вида `https://script.google.com/macros/s/.../exec`.
5. В репозитории откройте `config.js` и вставьте URL в `CONFIG.API_URL`.

> После изменений в `Code.gs` создайте **новое развертывание** (или «Управление развертываниями» → изменить версию), иначе сайт будет вызывать старую версию.

### 4. GitHub Pages

1. Создайте репозиторий на GitHub и загрузите файлы (кроме `Code.gs` — он живёт только в Apps Script, но можно оставить в репо как эталон).
2. **Settings → Pages**:
   - Source: **Deploy from a branch**
   - Branch: `main` / папка `/ (root)`
3. Откройте сайт: `https://<username>.github.io/<repo>/`

Перед публикацией убедитесь, что в `config.js` указан рабочий `API_URL`.

## Возможности

- Форма: имя, телефон, комментарий → **Сохранить**
- Таблица заявок с датой создания
- **Удалить** — удаление из интерфейса и из таблицы
- **Поиск по имени** без перезагрузки страницы
- Адаптивная вёрстка для мобильных

## API (Apps Script)

Запросы с фронтенда — **`GET`** с query-параметрами (POST из браузера к Apps Script часто ломается из‑за редиректа 302).

| `action` | Поля | Описание |
|----------|------|----------|
| `list` | — | Список заявок |
| `create` | `name`, `phone`, `comment` | Новая заявка + Telegram |
| `delete` | `id` | Удаление по UUID |

Резервный вариант через GET (тот же URL):

- `?action=list`
- `?action=create&name=...&phone=...&comment=...`
- `?action=delete&id=...`

## Устранение неполадок

| Проблема | Решение |
|---------|---------|
| «Укажите URL в config.js» | Замените `YOUR_SCRIPT_ID` на URL развертывания |
| Пустая таблица / ошибка загрузки | Проверьте доступ «Все», выполните `setupSheet`, обновите развертывание |
| Нет сообщений в Telegram | Проверьте Script Properties, что бот добавлен в чат/группу |
| CORS / сеть | Используется `text/plain` POST; URL должен заканчиваться на `/exec` |
